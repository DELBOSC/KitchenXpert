# KitchenXpert Nginx - Deployment Checklist

## Pre-Deployment

### Development Environment

- [ ] Nginx installed (`nginx -v`)
- [ ] Backend API running on port 4000
- [ ] Frontend built and ready to deploy
- [ ] Local hosts file updated
- [ ] Self-signed certificates generated

### Production Environment

- [ ] Server/VPS provisioned
- [ ] Domain names registered and DNS configured
- [ ] Firewall configured (ports 80, 443 open)
- [ ] SSL certificates obtained (Let's Encrypt)
- [ ] DH parameters generated (4096-bit)
- [ ] Backend services configured and running
- [ ] Frontend applications built
- [ ] Monitoring tools installed

## Configuration

### Main Configuration (nginx.conf)

- [ ] Worker processes set to `auto`
- [ ] Worker connections set appropriately (1024+)
- [ ] Gzip compression enabled
- [ ] Security headers configured
- [ ] SSL protocols limited to TLS 1.2+
- [ ] Rate limiting zones defined
- [ ] Upstream backends configured
- [ ] Cache directories defined
- [ ] Environment-specific config included (dev/prod)

### Environment Configuration

#### Development (nginx.dev.conf)

- [ ] Debug logging enabled
- [ ] HTTP-only mode configured
- [ ] CORS set to permissive
- [ ] Caching disabled
- [ ] Hot reload support configured

#### Production (nginx.prod.conf)

- [ ] HTTPS enforced
- [ ] HTTP/2 enabled
- [ ] HSTS headers enabled (2-year max-age)
- [ ] Strict CSP configured
- [ ] Aggressive caching enabled
- [ ] Rate limiting enforced
- [ ] Security headers maximized
- [ ] Brotli compression enabled (if available)

### Virtual Hosts

- [ ] Server name updated (`api.yourdomain.com`)

- [ ] Rate limiting applied
- [ ] CORS origins restricted
- [ ] Health check endpoint working
- [ ] Metrics endpoint restricted
- [ ] File upload size limits set
- [ ] GraphQL endpoint configured (if used)

#### Frontend App (app.conf)

- [ ] Server name updated (`yourdomain.com`)
- [ ] SSL certificates configured
- [ ] Document root correct (`/var/www/frontend/dist`)

#### Partner Portal (partner-portal.conf)

- [ ] Server name updated (`partners.yourdomain.com`)
- [ ] SSL certificates configured
- [ ] HTTPS enforced (HTTP redirects)
- [ ] Enhanced security headers enabled

- [ ] Rate limiting stricter than main app

#### Documentation (documentation.conf)

- [ ] Server name updated (`docs.yourdomain.com`)
- [ ] SSL certificates configured

## Directories & Permissions

### Web Roots

- [ ] `/var/www/frontend/dist` created
- [ ] `/var/www/partner-portal/dist` created
- [ ] `/var/www/docs` created
- [ ] Correct ownership (nginx:nginx or www-data:www-data)
- [ ] Correct permissions (755 for dirs, 644 for files)

### Cache Directories

- [ ] `/var/cache/nginx/api` created
- [ ] `/var/cache/nginx/static` created
- [ ] Owned by nginx user
- [ ] Writable by nginx

### SSL Certificates

- [ ] Certificate files present (.crt)
- [ ] Private key files present (.key)

## Rollback Plan

### If Issues Occur

- [ ] Backup location documented: `/etc/nginx/backup_YYYYMMDD_HHMMSS`
- [ ] Rollback procedure tested
- [ ] Downtime window communicated
- [ ] Alternative configurations ready

### Rollback Steps

1. Stop Nginx: `sudo systemctl stop nginx`
2. Restore backup: `sudo cp -r /backup/* /etc/nginx/`
3. Test config: `sudo nginx -t`
4. Start Nginx: `sudo systemctl start nginx`
5. Verify services: Check all endpoints
