# KitchenXpert - Présentation Technique Développeur

---

## PARTIE 1 : Genèse et Vision du Projet

### Le Problème

Concevoir une cuisine est un processus complexe, coûteux et frustrant :

- **Pour le particulier** : il navigue entre dizaines de marques (IKEA, Schmidt,
  Leroy Merlin, Castorama...), sans savoir si les meubles sont compatibles entre
  eux, si les dimensions rentrent dans sa pièce, ou si son budget est réaliste.
- **Pour le professionnel (cuisiniste)** : il jongle entre logiciels
  propriétaires, catalogues papier, et outils 3D vieillissants. Chaque marque a
  son propre outil, aucun n'est interopérable.
- **Pour le fabricant** : il n'a pas de canal digital unifié pour toucher les
  clients finaux ET les professionnels.

### La Solution : KitchenXpert

Une **plateforme SaaS complète** qui unifie tout le parcours de conception de
cuisine :

1. **Questionnaire intelligent** : comprendre les besoins (espace, style,
   budget, habitudes culinaires)
2. **Générateur IA** : proposer des configurations optimisées automatiquement
3. **Designer 3D interactif** : visualiser et modifier sa cuisine en temps réel
4. **Catalogue multi-fournisseurs** : 183 fournisseurs intégrés, produits réels
   avec prix
5. **Commande et paiement** : du design au devis, du devis à la commande
6. **Portail partenaires B2B** : interface dédiée pour les professionnels

### Les Utilisateurs Cibles

| Rôle                       | Usage                                                                    |
| -------------------------- | ------------------------------------------------------------------------ |
| **Particulier**            | Concevoir sa cuisine, explorer le catalogue, commander                   |
| **Cuisiniste / Pro**       | Gérer ses projets clients, accéder aux catalogues pro, générer des devis |
| **Admin**                  | Gérer les utilisateurs, les rôles, auditer l'activité                    |
| **Partenaire (fabricant)** | Synchroniser son catalogue, suivre les commandes, analytics              |

## PARTIE 1B : Pourquoi KitchenXpert Surpasse ses Concurrents

### Le Paysage Actuel — et ses Limites

Aujourd'hui, un particulier qui veut rénover sa cuisine a ces options :

| Outil / Service                      | Ce qu'il propose                                      | Ce qu'il ne fait PAS                                                |
| ------------------------------------ | ----------------------------------------------------- | ------------------------------------------------------------------- |
| **IKEA Kitchen Planner**             | Planification 2D/3D simple, catalogue IKEA uniquement | Autres marques, pas d'IA, pas de VR, pas de devis certifié          |
| **Schmidt / Mobalpa / Cuisinella**   | Logiciel propriétaire, une seule marque               | Interopérabilité zéro, nécessite RDV en magasin                     |
| **Leroy Merlin / Castorama planner** | Plan 2D basique, catalogue limité                     | Pas de 3D, pas d'IA, pas de VR/AR                                   |
| **Homebyme / RoomSketcher**          | Planificateur générique polyvalent                    | Non spécialisé cuisine, pas d'accords fournisseurs, pas de commande |
| **Houzz / Pinterest**                | Inspiration visuelle                                  | Aucune fonctionnalité de planification ou de commande               |
| **AutoCAD / SketchUp**               | Modélisation 3D professionnelle                       | Courbe d'apprentissage énorme, licences coûteuses, zéro catalogue   |
| **Architecte d'intérieur**           | Conseil personnalisé                                  | 1 500 €–5 000 € de prestation, délais de semaines                   |

**Le résultat** : l'utilisateur passe en moyenne 4 à 6 semaines à naviguer entre
ces outils, refaire ses plans à chaque changement de fournisseur, et ne sait
jamais si sa cuisine tient dans son budget avant d'avoir signé.

---

### Ce que KitchenXpert fait différemment

#### 1. Toutes les marques dans un seul outil

IKEA, Schmidt, Mobalpa, Nobilia, Bosch, Siemens, AEG, KitchenAid, Miele...
KitchenXpert aggrège 80+ fournisseurs (meubles + électroménager) dans une
interface unique. L'utilisateur compare des meubles IKEA avec un plan de travail
Dekton et un four Bosch — sans changer d'onglet.

#### 2. L'IA fait le travail à sa place

Au lieu de passer des heures à placer des meubles un à un, l'utilisateur répond
à un **questionnaire de 4 minutes** (espace, style, budget, habitudes de
cuisine). L'IA génère plusieurs **configurations complètes et optimisées** :
meubles positionés, électroménager choisi, budget calculé, ergonomie vérifiée.
Il ne reste plus qu'à ajuster.

#### 3. Voir avant d'acheter — vraiment

- **Vue 3D interactive** : tourner autour de la cuisine, changer les matériaux
  en temps réel, modifier l'éclairage
- **Réalité virtuelle** : enfiler un casque (Meta Quest, HTC Vive) et se
  promener dans sa future cuisine comme si elle était construite
- **Réalité augmentée** : pointer son téléphone vers sa cuisine actuelle et voir
  les nouveaux meubles superposés dessus, dans les bonnes dimensions, dans la
  vraie pièce

#### 4. Du design à la commande, sans rupture

Le parcours est **entièrement intégré** : questionnaire → design IA → 3D → devis
certifié → e-signature → paiement → mise en relation avec un installateur.
Aucune ressaisie, aucun copier-coller entre outils.

#### 5. Des outils professionnels pour tous

Des fonctionnalités réservées aux cuisinistes professionnels sont accessibles à
tous :

- **Vérification de conformité** réglementaire (normes EN 1717, NF DTU 60.1)
- **Simulateur de flux de travail** : le triangle de travail est-il optimisé ?
  Combien de mètres parcourt-on pour préparer un repas ?
- **Devis certifié avec e-signature** conforme eIDAS
- **Suivi des prix** avec alertes quand un produit baisse

#### 6. L'environnement et le budget, pas en dernier

- **Calculateur de financement** avec les aides réelles (MaPrimeRénov, CEE, TVA
  5.5%)
- **Empreinte carbone** calculée pour chaque configuration (meubles +
  électroménager + livraison)
- **Suivi des prix en temps réel** avec historique et alertes personnalisées

## PARTIE 1C : Ce Qu'un Utilisateur Peut Faire — Parcours Complet

> Cette section est écrite en langage simple. Un développeur qui rejoint le
> projet doit comprendre exactement ce qu'il construit, du point de vue de
> l'utilisateur final.

### Le Parcours Type d'un Particulier

```
[1] Je m'inscris
    → Email + prénom + nom + mot de passe
    → Vérification email (lien reçu par mail)
    → Je suis connecté

[2] (Optionnel) Je scanne ma pièce avec mon téléphone
    → J'ouvre le Photo Room Scanner (PhotoScanPanel dans le Designer)
    → Étape 0 — Guide visuel : SVG interactif vue de dessus avec 3 positions caméra
      recommandées (coin arrière, seuil de porte, côté opposé) avant tout upload
    → Je prends 1 à 3 photos de ma cuisine actuelle sous différents angles
       ↳ Glisser-déposer ou bouton "Parcourir" → JPEG/PNG/WebP, max 10 MB chacune
       ↳ Prévisualisation immédiate avec miniatures
       ↳ Badge qualité automatique : "⚠ Résolution faible" si < 800×600 px
    → L'IA (Claude Vision) analyse les photos et détecte automatiquement :
       ↳ Les dimensions de la pièce (longueur, largeur, hauteur en mètres)
       ↳ Les murs avec leur longueur et présence de fenêtres/portes
       ↳ Les ouvertures (portes, fenêtres avec dimensions estimées)
       ↳ Les points techniques (prises électriques, interrupteurs, arrivées d'eau/gaz)
       ↳ Les obstacles (poutres, recoins, radiateurs)
    → Résultats : plan SVG interactif du sol (murs épais, arcs de porte, hachures fenêtre,
      icônes prises/eau avec tooltip, obstacles, cotes dimensionnelles sur chaque mur)
    → Dimensions éditables inline avant d'appliquer (inputs number pré-remplis par l'IA)
    → Le plan au sol est généré automatiquement et pré-rempli dans le questionnaire
    → Si l'appareil a un capteur LiDAR (iPhone Pro, iPad Pro) : scan de précision
      millimétrique via vraie boucle WebXR depth-sensing (accumulation de frames à 60 fps,
      échantillonnage 3×3 par frame, médiane par direction, calcul dimensions réelles)
      Heatmap de profondeur en temps réel sur le flux caméra (rouge=proche, bleu=lointain)
      Badge de méthode affiché : "LiDAR" / "Photo" / "Manuel"
      Sinon, fallback sur analyse photo (même pipeline)
    → Confiance IA affichée par élément (dimensions, ouvertures, points techniques)

[3] Je réponds au questionnaire (4 min — dimensions pré-remplies si scan fait)
    → Étape 1 — Mon profil : je cuisine souvent ? seul ou en famille ?
    → Étape 2 — Mon espace : dimensions de la pièce (pré-remplies si scan), forme
    → Étape 3 — Mon style : moderne, scandinave, industriel, campagne, minimaliste ?
    → Étape 4 — Mon budget : quel est mon budget total et ma tolérance ?

[4] L'IA génère mes configurations
    → 3 à 5 propositions complètes s'affichent
    → Chaque proposition inclut : plan de cuisine, liste de meubles avec prix,
      liste d'électroménager recommandé, score d'ergonomie, total budget
    → Je sélectionne celle qui me plait le plus

[5] J'ouvre le Designer 3D
    → Je vois ma cuisine en 3D dans mon navigateur, sans installer quoi que ce soit
    → Je peux (la SOURIS est l'outil principal) :
       ↳ Clic gauche sur un meuble → le sélectionner (surbrillance bleue)
       ↳ Shift + Clic → multi-sélection
       ↳ Clic-drag sur un meuble sélectionné → le déplacer dans la pièce
       ↳ W / E / R changent le mode (déplacer / pivoter / redimensionner),
         puis la souris agit sur les gizmos Three.js
       ↳ Clic-drag bouton droit ou milieu → orbiter la caméra autour de la scène
       ↳ Molette → zoomer/dézoomer (de 1m à 50m de distance)
       ↳ Glisser-déposer un produit depuis le catalogue → ghost bleu de
         prévisualisation pendant le drag, pose au relâchement
       ↳ Mode mesure : 2 clics → tracé d'une mesure avec label en cm
       ↳ Mode walkthrough : mouvements souris → regarder autour à hauteur d'œil
         (pointer lock, yaw/pitch limité à ±85°, collision avec les murs)
    → Fonctionnalités additionnelles :
       ↳ Snap automatique pendant le drag : murs, grille 1mm, alignement objets,
         face-à-face, hauteurs standards (850mm plan de travail)
       ↳ Dimensions L × H × P affichées en temps réel pendant la transformation
       ↳ Changer les matériaux des façades (bois, laque, béton, verre...)
       ↳ Changer les plans de travail (granit, quartz, inox, bois...)
       ↳ Changer l'éclairage (neutre, chaud, froid, naturel)
       ↳ Voir le plan du dessus ou une vue en élévation (touche T)
       ↳ Prix total mis à jour en temps réel à chaque modification
       ↳ Ctrl+Z / Ctrl+Y → annuler / rétablir chaque action (undo/redo complet)
       ↳ Ctrl+S → sauvegarder l'état
    → J'importe une photo d'inspiration pour en extraire le style (StyleTransferModal) :
       ↳ Mode "Fichier" : glisser-déposer ou "Parcourir" une photo de magazine, Pinterest...
       ↳ Mode "URL" (nouveau) : coller une URL d'image → prévisualisation immédiate + analyse
       ↳ L'IA (Claude Vision) analyse et extrait automatiquement :
         – Style détecté (modern, traditional, farmhouse, Scandinavian, industrial...)
         – Palette de couleurs (primaire, secondaire, accent, neutre) → swatches 80×80 px
           avec nom de couleur en français au survol (ex: "Beige chaud", "Gris ardoise")
           et bouton copier le code hex en presse-papier
         – Matériaux : façades, plan de travail, crédence, sol → vignette CSS texturée
           (gradient bois, marbre, métal, béton selon le matériau détecté)
         – Style des portes et poignées
         – Marques de l'univers correspondant suggérées
       ↳ Application sélective (7 checkboxes) : choisir exactement ce qu'on applique
         (palette, façades, plan de travail, crédence, style portes, poignées, sol)
       ↳ "Appliquer la sélection" → seules les propriétés cochées sont transmises à la scène

[6] Je visualise en immersion totale
    → Mode VR : si j'ai un casque (Meta Quest, HTC Vive, PlayStation VR),
      je me retrouve dans ma cuisine et je peux me déplacer dedans
    → Mode AR : avec mon téléphone, je pointe vers ma cuisine actuelle
      et je vois ma future cuisine superposée dans la vraie pièce,
      avec les vraies dimensions, en temps réel

[7] Je vérifie l'ergonomie
    → Le simulateur de flux trace mes déplacements simulés (préparer un repas,
      nettoyer, servir) et calcule :
       ↳ Distance totale parcourue
       ↳ Score du triangle de travail (réfrigérateur → évier → plaque)
       ↳ Suggestions pour améliorer (ex: "déplacer le réfrigérateur de 40 cm")

[8] Je suis les prix et j'attends le bon moment
    → J'ajoute des produits en liste de suivi
    → Je configure une alerte : "Préviens-moi si ce meuble IKEA descend sous 180 €"
    → Je reçois une notification quand le prix change
    → Je vois l'historique des prix sur un graphique

[9] Je planifie mes appareils connectés
    → Je place mes appareils domotiques sur le plan (hotte connectée, frigo IoT,
      prises intelligentes, détecteur de fumée Matter...)
    → Je vois la couverture WiFi/Zigbee dans la cuisine
    → Je crée des scènes d'automatisation ("Départ maison" → tout éteint)
    → Je vérifie la compatibilité entre marques (Matter 1.3)

[10] Je calcule le financement
    → Je saisis le montant, la durée, le taux → tableau d'amortissement
    → Je vois les aides disponibles : MaPrimeRénov, CEE, TVA 5.5%
    → Je vois le reste à charge réel après déduction des aides
    → Je compare plusieurs organismes de crédit

[11] Je commande
    → Je génère un devis certifié depuis le designer
    → Le devis liste tous les produits avec prix, références fournisseurs, délais
    → Je signe électroniquement (e-signature conforme eIDAS)
    → Le devis est horodaté et archivé légalement (10 ans)
    → Je passe commande directement depuis la plateforme (Stripe)

[12] Je trouve un installateur
    → Je cherche par code postal ou ville
    → Je filtre par spécialité (pose, plomberie, électricité, carrelage)
    → Je vois les notes, les avis clients, les certifications
    → Je demande un devis directement depuis la fiche

[13] Je mesure l'impact environnemental
    → La plateforme calcule le CO₂ de ma configuration :
       ↳ Fabrication des meubles (par matériau et marque)
       ↳ Énergie de l'électroménager sur 10 ans
       ↳ Transport / livraison estimé
    → Elle suggère des alternatives moins carbonées

[14] Je rénove une cuisine existante (RenovationPage)
    → J'importe une ou plusieurs photos de ma cuisine actuelle :
       ↳ Glisser-déposer ou bouton "Parcourir" (JPEG/PNG/WebP, max 10 MB)
       ↳ L'appareil photo est activable directement si sur mobile (capture="environment")
       ↳ Prévisualisation immédiate avant l'envoi
    → L'IA (Claude Vision) analyse chaque photo et retourne :
       ↳ État de chaque composant : meubles, plan de travail, électroménager, sol, murs
       ↳ Niveau d'usure (bon / passable / à remplacer / urgent)
       ↳ Éléments à conserver vs à remplacer
       ↳ Coût de démolition estimé en EUR
       ↳ Score de confiance de l'analyse (0-100%)
    → Priorisation des travaux : urgent / important / optionnel
    → Décomposition du budget de rénovation poste par poste
    → Slider visuel avant/après : ma photo actuelle vs la configuration générée
    → Taux de progression affiché pendant l'analyse (barre de progression)
```

### Ce qu'un Professionnel (Cuisiniste) Peut Faire en Plus

Un cuisiniste s'inscrit avec le rôle `professional` et obtient des accès étendus
:

- **Plusieurs projets clients** : chaque client a son propre projet, avec son
  questionnaire, ses cuisines, ses devis
- **Collaboration** : inviter un client à voir et commenter son projet (lien de
  partage avec ou sans mot de passe)
- **Vérification de conformité** : checker automatiquement que la cuisine
  respecte les normes professionnelles (EN 1717 eau, NF DTU 60.1 plomberie,
  etc.)
- **Devis certifiés en série** : générer et faire signer des devis pour chaque
  client, les archiver légalement
- **Catalogue pro** : accès aux catalogues fournisseurs pro (prix remisés,
  références spécifiques)
- **Accès au portail partenaires** : si le pro est aussi fabricant, il gère son
  catalogue depuis le portail dédié

---

### Ce qu'un Fabricant (Partenaire) Peut Faire

Un fabricant (ex: Schmidt, Nobilia, Bosch) accède au **portail partenaires**
(`/partner/*`) :

- **Dashboard en temps réel** : combien de ses produits sont en catalogue,
  combien de vues, combien de commandes ce mois, revenus générés
- **Gestion du catalogue** : ajouter, modifier, désactiver ses produits
  directement
- **Suivi des commandes** : voir toutes les commandes contenant ses produits,
  mettre à jour les statuts (confirmé, expédié, livré)
- **Analytics de ventes** : CA journalier/mensuel/annuel, top produits,
  évolution dans le temps
- **Accès séparé** : le portail partenaires est une application React distincte
  (`packages/partner-portal`), avec son propre design et ses propres routes
  backend

### Authentification

```
Utilisateur ──POST /auth/login──▶ Backend
                                    │
                          Vérifie email + bcrypt (12 rounds)
                                    │
                          Génère JWT access (15min) + refresh (7j)
                                    │
                    ◀── Set-Cookie: httpOnly, Secure ──
                                    │
Navigateur envoie cookies auto ──▶  Middleware authenticate()
                                    │
                          Vérifie JWT + blacklist
                          Injecte req.user = { userId, email, role }
                                    │
                          Vérifie ownership : entity.userId === req.user.userId
```

- Tokens stockés en **cookies httpOnly** (jamais en localStorage, ni côté
  frontend ni côté partner-portal)
- Refresh automatique côté frontend (401 → refresh → retry)
- Blacklisting des tokens à la déconnexion
- **Tokens email invalidés par `deleteMany`** (suppression atomique, prévention
  race condition/réutilisation)
- Rate limiting : 5 tentatives / 15 min sur `/auth`, incluant
  `password-reset/confirm`
- Pas de fallback JWT en dur dans la configuration (erreur si variable
  manquante)
- Vérification que le session secret n'est pas le défaut en production
- Handlers globaux : `process.on('unhandledRejection')`,
  `process.on('uncaughtException')`, `SIGTERM`, `SIGINT`

### 3.2 Sécurité — 14 middlewares + protections renforcées

| Middleware             | Fonction                                                                             |

| `sanitize`             | Prévention XSS                                                                       |
| `cors`                 | Configuration CORS (origines restreintes en production, plus de wildcard)            |
| `validation`           | Validation Zod sur body/query/params (AI generator + questionnaire)                  |
| `error`                | Gestion globale des erreurs                                                          |
| `logging`              | Log des requêtes/réponses                                                            |
| `monitoring`           | Métriques de santé                                                                   |
| `audit`                | Capture des événements d'audit                                                       |

**Protections ajoutées lors des cycles 7 et 8** :

- **Path traversal upload** : whitelist `ALLOWED_FOLDERS` — rejet de tout chemin
  hors liste
- **Vérification share link** : mot de passe de partage vérifié avec
  `bcrypt.compare()` (était ignoré)
- **IDOR Stripe** : vérification `customer.metadata?.userId === userId` sur
  historique paiements ET sur intent/cancel
- **Upload ownership** : `getFile`, `getFileMetadata`, `listFiles`, `copyFile`
  vérifient le propriétaire
- **Tokens email atomiques** : `deleteMany` à la place de `updateMany` pour
  prévenir les race conditions de réutilisation
- **Handlers process** : `unhandledRejection`, `uncaughtException`, `SIGTERM`,
  `SIGINT` pour éviter les crashs silencieux
- **Rate limiting étendu** : `password-reset/confirm`, endpoints monitoring
  (écriture), endpoints partage public
- **Validation Zod** : schémas ajoutés sur routes AI generator, questionnaire,
  admin bulk
- **CORS production** : refus des requêtes sans `Origin` en production
- **Session secret** : vérification que le secret n'est pas le défaut en
  production
- **Scraper** : authentification API key ajoutée (middleware `requireApiKey`),
  CORS restreint, SHA-256 (plus MD5)

### Conventions à Respecter

- **`userId`**, jamais `id`, dans `req.user` (JWTPayload :
  `{ userId, email, role }`)
- **Prisma singleton** : `import { prisma } from '../../database/client'`
  (jamais `new PrismaClient()`)
- **Ownership** : toujours vérifier
  `entity.userId === req.user?.userId || req.user?.role === 'admin'`
- **Stripe IDOR** : toujours vérifier `customer.metadata?.userId === userId` +
  check `!metadata?.userId` → 400 en amont
- **Upload path traversal** : valider le dossier contre `ALLOWED_FOLDERS`
  whitelist
- **Email tokens** : utiliser `deleteMany` (atomique) — pas `updateMany` (race
  condition)
- **Frontend fetch** : `credentials: 'include'` obligatoire, AbortController
  dans les useEffect
- **URLs API** : utiliser `API_ENDPOINTS.*` — jamais de chaîne hardcodée
- **Erreurs** : utiliser `getErrorMessage(err, defaultMessage)` depuis
  `utils/error-handling.ts`
- **Prisma JSON** : caster avec `as Prisma.InputJsonValue` (pas `as any`)
- **Pas de** `window.location.reload()` → utiliser le pattern retryCount
- **Pas de** `window.confirm()` → utiliser des modales React
- **Pas de** `window.location.href` → utiliser la navigation React (useNavigate)
- **Pas de** `Math.random()` pour les IDs → `crypto.randomBytes()` ou
  `crypto.randomUUID()`
- **Pas de** `Record<string, any>` → utiliser `Record<string, unknown>`
- **Pas de** `catch (error: any)` → type narrowing :
  `error instanceof Error ? error.message : 'Unknown error'`
- **Pas de** `localStorage` pour les tokens → cookies httpOnly exclusivement
- **pnpm** : version 8.15.0 exacte (pas `@latest`), installer via `corepack`
- **JWT** : variables `JWT_ACCESS_SECRET` / `JWT_REFRESH_SECRET` (pas
  `JWT_SECRET`)
- **Node.js** : version 20 (voir `.nvmrc` et `.node-version`)
- **PostgreSQL** : version 16
- **Vite env vars** : préfixe `VITE_*` (pas `REACT_APP_*`)
- **i18n** : utiliser `t('namespace.key', 'Fallback text')` sur toutes les
  pages, `i18n.language` pour la locale
- **Modales** : `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
- **Tables de données** : `aria-busy={isLoading}` sur le conteneur pendant le
  chargement
- **FormData** : vérifier avec `instanceof FormData` avant `JSON.stringify` dans
  l'API client
- **3D Engine** : toujours appeler `dispose()` sur géométries, matériaux et
  textures créés dynamiquement
- **URL.createObjectURL** : révoquer via
  `requestAnimationFrame(() => setTimeout(() => URL.revokeObjectURL(url), 0))`
- **ANTHROPIC_API_KEY** : requis pour les features d'enrichissement IA — un
  avertissement est affiché au démarrage si absent
